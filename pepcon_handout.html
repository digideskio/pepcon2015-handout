HTML5 Publishing Resources

HTMLBook Project
Open source resources for using HTML5 as a book authoring/production format: XML Schema, CSS stylesheets for print book design, and XSLT-based tools for producing ebook and Web outputs.
HTMLBook Specification
Latest Unofficial Working Draft specification and content model for HTMLBook.
W3C HTML5 Specification
Official W3C specification for HTML5 and XHTML5.
HTML5 for Publishers (O'Reilly, 2011)
Free ebook by Sanders Kleinfeld on authoring next-generation interactive/multimedia ebook content in HTML5. Covers canvas, audio, video, MathML, and SVG.

CSS Resources for print books

CSS Paged Media Level 3
W3C specification for CSS Paged Media, which provides a CSS styling/typesetting module geared toward PDF and print outputs.
CSS Generated Content for Paged Media Module
W3C specification for autogenerated text with Paged Media, for producing running headers/footers, cross-references, footnotes, and more using CSS.
“Building Books with CSS3”
Comprehensive introductory tutorial on developing print-book designs using CSS Paged Media by Nellie McKesson.
“CSS for Print”
Four-part screencast series by Nellie McKesson that both covers the basics of CSS stylesheet development (from selectors to the box model) and provides a walkthrough of Paged Media's features for print compositing with CSS.

CSS print typesetting engines

Antenna House Formatter
Commercial typesetting engine for producing professional print books using XHTML5 and CSS3. Supports CSS3 Paged Media, as well as some proprietary CSS extensions.
Prince
Commercial typesetting engine for producing professional print books using XHTML5 and CSS3. Supports CSS3 Paged Media; also supports DOM manipulation via embedded JavaScript in script elements.
Vivliostyle
(Currently in Alpha): Open source JavaScript engine for typesetting PDF in the browser. Limited CSS Paged Media support at this time. Also supports typesetting via EPUB Adaptive Layout.

Git Basics Cheat Sheet

git clone

Download a copy of a remote Git repository to your local machine:

$ git clone git@github.com:oreillymedia/HTMLBook.git

git pull

Update your local repository with the latest changes from the corresponding remote Git repository:

$ git pull origin master

git add

Schedule the addition of new file(s) or updates to existing files, which will be saved with the next git commit.

$ git add new_file.html

git rm

Schedule file(s) to be deleted with the next git commit (their history is still preserved)

$ git rm file_i_want_deleted.html

git commit

Save additions and changes made with git add (or deletions made with git rm), with a log message of your choice

$ git commit -m'Made proofread edits in chapters 1, 2, and 3.'

git push

Upload changes you've made in your local repo to the corresponding remote Git repository:

$ git push origin master

git log

See a log showing the revision history of any file, including when changes were made, who made them, and the corresponding log messages.

$ git log ch10.html

jQuery Cheat Sheet

HTML file setup

Include script elements for both the jQuery library and your custom JavaScript in the head:

<html>
  <head>
    <title>My Interactive eBook</title>
    <script src="jquery-1.11.3.min.js"></script>
    <script src="my_custom_jquery.js"></script>
    ...

Custom jQuery skeleton

Start with the following in your custom jQuery script file:

$(document).ready(function() {
  // Your custom code will go here
});

Selecting elements

By tag name:

$("h2")

By class:

$(".heading")

By tag name and class:

$("h2.heading

By id:

$("#footnotes")

By parent tag name and tag name:

$("section h2")

Showing/hiding elements

Use show() to turn on display of selected element(s):

$(".sidebar").show();

And hide() to turn off display of selected elements:

$(".sidebar").hide();

Binding event handlers

Use on() to define behavior when a specified event is triggered on selected element(s):

$("button").on("mouseover", function () {
   // Code here will run when user mouses over button elements
});

Dynamically applying CSS

Use css() on selected element(s) to apply CSS rules:

$("button").on("mouseover", function () {
   // Change background color of buttons when user mouses over them
   $(this).css("background-color", "red");
});
